## 工作流程（6阶段）
### 阶段0：需求分析与主题解构（10分钟）

**模型：glm-5**（规划需要深度推理和任务拆解能力）

**编排组件**：`scripts/orchestration/executor.py` → `GodtierExecutor`

**步骤1：理解研究主题**
- 明确分析对象（公司/行业/策略/技术）
- 识别核心问题（Why/How/What）
- 确定分析模式（full/quick/risk/compete）

**步骤2：初始化编排器**

```python
from scripts.orchestration.executor import GodtierExecutor

executor = GodtierExecutor(topic="研究主题", mode="full")
plan = executor.phase0_analyze(user_request="用户的原始请求")
# plan 包含：topic, mode, core_questions, agent配置, 时间预算
```

**步骤3：设计Agent方案**

根据主题复杂度动态调整Agent数量：

| 模式 | 数据采集Agent | 分析Agent | 预计耗时 | 适用场景 |
|------|-------------|-----------|---------|----------|
| full | 8个 | 7个 | ~2小时 | 完整13层分析 |
| quick | 4个 | 3个 | ~30分钟 | L-1+L0+L1+L10精简版 |
| risk | 5个 | 4个 | ~1小时 | L-1+L0+L6+L7+L10风险评估 |
| compete | 6个 | 5个 | ~1小时 | L3+L4+L5+L6竞争分析 |

**步骤4：向用户展示规划并等待批准**

以下是要展示给用户的完整规划模板（直接复制使用）：

```
我分析了这个任务，可以这样拆分：

阶段0：需求分析与主题解构（10分钟）
- 主题：[具体主题]
- 模式：[full/quick/risk/compete]
- 核心问题：[列出2-3个核心问题]

阶段1：多维度数据采集（[N]个Agent并行，40分钟）
- 金融数据Agent：股价、财报、估值（SEC EDGAR、投行研报）
- 技术动态Agent：论文、开源、产品（arXiv、GitHub）
- 政策监管Agent：法规、合规（Federal Register、SEC）
- 市场情报Agent：融资、并购、份额（CB Insights、PitchBook）
- 学术前沿Agent：研究、突破（NBER、顶级会议）
- 宏观趋势Agent：周期、地缘（Stratechery、a16z）
- 竞争情报Agent：对手、替代（行业报告）
- 历史类比Agent：相似事件、结果差异（NBER、FRED）

阶段2：深度分析（[N]个Agent并行，30分钟）
- Agent 0：L-1残差Alpha分析（误差扫描+共识误判+尾部误定价）
- Agent A：L0反向证伪 + L1范式谬误
- Agent B：L2系统约束 + L3激励机制
- Agent C：L4信息博弈 + L5微观操作
- Agent D：L6对手盘 + L7时间动态
- Agent E：L8行为心理 + L9定量模型（调用脚本）
- Agent F：L10终局推演

阶段3：文章撰写与整合（20分钟）
- 13层结构撰写
- 105个信源引用
- 残差Alpha洞察整合

阶段4：质量验证与幻觉检测（10分钟）
- 三层幻觉检测（数字级+逻辑级+全文级）
- 六维评分自评

阶段5：输出与存档（5分钟）
- Markdown + HTML + PDF
- 发送到Discord
- 审计日志存档

预计总耗时：约[2小时/30分钟/1小时]
预计产出：10000-15000字深度分析文章
信源覆盖：20-40个顶级信源

这样可以吗？需要调整吗？
```

**步骤5：等待用户批准**
- 绝对不要直接启动Agent
- 等待用户确认或调整
- 用户批准后调用 `executor.phase0_approve()`

```python
# 用户批准后
executor.phase0_approve(plan)
# 自动保存 research_plan.json 到 analysis_dir
# 自动完成阶段0门验证
# 准备进入阶段1
```

## General Mode — Agent Planning Template

当用户使用通用模式（`mode: general`）时，将以下规划模板展示给用户：

```
我分析了这个热点话题，可以这样拆分：

阶段0：需求分析与主题解构（10分钟）
- 主题：[具体热点话题]
- 模式：general（通用深度分析）
- 核心问题：[列出2-3个核心问题]

阶段1：多维度数据采集（8个Agent并行，40分钟）
- 事件基础数据 Agent：时间线、规模、参与者、关键事实
- 技术/科学视角 Agent：相关技术背景、科学原理
- 政策/法规/治理 Agent：法律法规、政策公告
- 行业/商业动态 Agent：产业链影响、市场变化
- 社会舆论/公众情绪 Agent：社交媒体情绪、公众意见
- 宏观背景/历史脉络 Agent：历史背景、地缘政治背景
- 竞争/替代视角 Agent：对立观点、边缘化视角
- 跨领域类比 Agent：其他领域的相似模式、跨学科联系

阶段2：深度分析（7个Agent并行，30分钟）
- Agent 0：盲点检测（主流叙事忽略了什么？）
- Agent A：前提挑战 + 叙事标签标注
- Agent B：约束分析（法律/经济/技术/时间约束）
- Agent C：信息博弈（信息不对称 + 微观机制）
- Agent D：利益相关者分析（赢家/输家 + 系统风险）
- Agent E：行为心理 + 数据验证
- Agent F：终局推演（三情景分析）

阶段3：文章撰写与整合（20分钟）
- 11段通用文章结构撰写
- 全网信源引用
- 盲点洞察整合

阶段4：质量验证与幻觉检测（10分钟）
- 三层幻觉检测（数字级+逻辑级+全文级）
- 新六维评分自评

阶段5：输出与存档（5分钟）
- Markdown + HTML + PDF
- 发送到Discord
- 审计日志存档

预计总耗时：约2小时
预计产出：10000-15000字深度分析文章
信源覆盖：30-50个全网信源
分析框架：通用13层分析架构（盲点→利益相关者→终局推演）

这样可以吗？需要调整吗？
```

通用模式与财经模式的关键差异：
- **信源**：热搜平台（百度/微博/知乎/B站）+ 新闻媒体 + 学术数据库 + 国际媒体
- **分析框架**：去金融化，保留逻辑骨架（博弈→利益相关者，定量→数据验证，终局推演→三情景）
- **评分维度**：信源多样性、逻辑严谨性、跨领域覆盖、时间广度、叙事校准、可验证性
- **计算脚本**：仅保留通用脚本（统计/回归/排序/清洗/验证），移除金融专用脚本（DCF/WACC/蒙特卡洛）

### 阶段1：多维度数据采集（5-8个Agent并行，40分钟）

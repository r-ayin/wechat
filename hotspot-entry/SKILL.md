### 🚀 自动模式

**行为**：用户给选题 → 直接跑管线，不提问。
**适用**：用户已经很清楚要做什么，只需要执行。

## Phase 0.5: Dankoe 需求采访（🎯 手动模式专用）

> 采访 prompt 模板：[prompts/dankoe-interview.md](../prompts/dankoe-interview.md)（微信版）
> 方法论来源：Dan Koe 两步法 + HUMAN 3.0 四象限

### 采访八问

按顺序提问，每次 1-2 问，根据回答动态深入。
**⑦⑧ 不直接提问框架名——AI 根据前六问在内部完成分类，仅用自然语言向用户确认。**

| # | 维度 | 核心问题 | 产出 | 下游消费者 |
|---|------|---------|------|-----------|
| ① | **受众定位** | 这期内容给谁看？他们现在对这个话题的认知停在哪个层面？ | 目标受众画像 | Phase 3 persona（语言复杂度锚定） |
| ② | **个人关联** | 你和这个话题有什么独特的关联？（经历/观察/立场）别人也可以写这个话题，但你写和别人写有什么不同？ | 独特视角来源 | Phase 3 persona（独特视角注入） |
| ③ | **核心误解** | 关于这个话题，最普遍的错误认知是什么？你每次听到都想反驳的是什么？ | 反常识锚点 | Phase 2 godtier（重点反驳方向）、Phase 3.5 QA（逻辑审查重点） |
| ④ | **情感锚点** | 你希望受众看完后感受到什么？愤怒？被理解？紧迫？清醒？还是某种复杂情绪？ | 情感基调 | Phase 3 persona（STYLE.md 情感温度参数） |
| ⑤ | **行动导向** | 你希望他们看完后做什么？或者停止做什么？如果受众做了一个具体行动，这期内容就成功了——那个行动是什么？ | CTA 方向 | Phase 3 persona（结尾设计） |
| ⑥ | **非共识角度** | 其他人都在怎么说这个话题？你的角度哪里不同——甚至相反？ | 差异化切口 | Phase 0 对标发现（差异化空间）、Phase 2 godtier（差异化分析） |
| ⑦ | **内容象限**（AI 静默分类） | AI 根据前六问判断选题落在哪个象限（内在个人/外在个人/内在集体/外在集体），用自然语言确认："所以这个选题主要是在帮人重新理解自己和世界的关系，对吗？"——**不提框架名** | 象限定位 | Phase 2 godtier（类比维度选择） |
| ⑧ | **认知层级**（AI 静默判断） | AI 根据前六问判断目标认知层级（1D信息→5D元认知），用自然语言确认："这个话题你希望讲多深？是让大家知道发生了什么的程度，还是给他们一个能自己分析类似问题的框架？"——**不提框架名** | 深度定位 | Phase 2 godtier（研究深度参数）、Phase 3 persona（写作复杂度） |

## Phase 0: 对标发现与热点扫描

### 0.1 找对标账号

**"三选三不选"原则** (DeepResearch 5-2 vote 验证):

| 选 | 不选 |
|----|------|
| 近3个月起号成功 | 头部大V（粉丝红利不可复制） |
| 5000-50000粉丝 | 高粉账号（时机窗口关闭） |
| 内容垂直度高 | 长期账号（增长模式不同） |

**五种发现方法：**

1. **站内搜索** — 领域关键词 → 用户标签 → 粉丝量±30%、近30天活跃
2. **热点宝观测** — 创作者服务中心 → 热点宝 → 账号观测 → 按涨粉速度排序
3. **巨量算数关联** — 主关键词 → 关联分析 → 衍生词 → 各自搜索TOP5
4. **高赞未爆反挖** ⭐ — 点赞3000-8000、粉丝<1万、15-45天内。这是发现潜力账号最有效的方法
5. **跨平台交叉** — 小红书/B站同关键词 → 收藏>5000 → 多平台作者

**账号分级：**

| 梯队 | 标准 | 数量 | 追踪频率 |
|------|------|------|---------|
| T1 核心 | 粉丝1-5倍、90天内有爆款、风格鲜明 | 3-5（默认值，根据竞品密度动态调整） | 每周 |
| T2 参考 | 粉丝相近、内容有差异化角度 | 5-8（默认值，根据竞品密度动态调整） | 每月 |
| T3 预警 | 新起号、增速快 | 不限 | 按需 |

**工具选择（冷启动全免费）：**
- 热点宝 + 巨量算数 + 站内搜索 → 0元
- 手动五维拆解 (20min/条) → 0元
- 飞瓜/蝉妈妈付费版 → 月产30条后升级

### 0.2 爆款拆解

**拆解框架（五维）：**

| 维度 | 拆什么 | 产出 |
|------|--------|------|
| 选题基因 | 普遍痛点 × 稀缺关注点 × 时机 | 选题模板 |
| 开头钩子 | 反常识/数字/身份/悬念/共鸣/宣言 | 钩子库 |
| 内容骨架 | 观点→反常识→证据→转折→新观点→CTA | 结构模板 |
| 节奏设计 | 0-3s/3-15s/15-25s/25-35s | 时长模板 |
| 差异化空间 | **竞品没做什么？** | 切入角度 |

**竞品爆款触发阈值：**
- 竞品视频 >10K点赞 → 24小时内完成拆解
- 竞品评论区出现"质量差/虚假"情绪 → 反定位机会
- 竞品进入新内容格式 → 立即跟进测试

### 0.3 热点扫描

**双轨扫描架构：**

| 轨道 | 工具 | 方法 | 更新频率 |
|------|------|------|---------|
| 🆕 自动扫描 | `topic-pool/hot-scanner.py` | 五支柱 × 43关键词 + 跨平台热榜 | 每 4h |
| 手动深挖 | 热点宝 + 巨量算数 + 评论区 | 行业热点 + 关联词云 + 竞品信号 | 每日 |

**自动扫描工作流：**

```
python topic-pool/hot-scanner.py scan → 生成查询列表
       │
       ▼
Claude WebSearch 逐条搜索 → results.json
       │
       ▼
python topic-pool/hot-scanner.py consume results.json --min-score 6.0 → 注入选题池
       │
       ▼
python topic-pool/hot-scanner.py report → 热点简报
```

**热点发现渠道（按获取难度）：**

| 渠道 | 方法 | 更新频率 |
|------|------|---------|
| 🆕 hot-scanner | 43 关键词覆盖五支柱 + 评分排序 | 每 4h |
| 热点宝 | 行业热点话题 + 涨粉排行 | 每日 |
| 巨量算数 | 关键词趋势 + 关联词云 | 每周 |
| 评论区挖掘 | T1竞品评论区高频提问词 → 未被满足的需求 | 每周 |
| 跨平台信号 | 小红书/知乎/B站同领域热门 | 每日 |
| 低粉爆款 | 搜索点赞3000-8000+粉丝<1万 | 每周 |

**热点质量判断：**

```
热点价值 = 普遍性(搜索量>100万?)
         × 稀缺性(同类视频<100条?)
         × 时效性(趋势上升中?)
         × 争议性(评论区有对立?)
         × 人格匹配度(与SOUL.md核心信念重叠?)
```

### 0.4 选题信号排序

**排序公式：**

```
选题得分 = UCB探索分 × 对标信号强度 × 人格匹配度
```

- **UCB探索分**: topic_pool.json 中的 bandit 算法分值，未做过的选题优先
- **对标信号强度**: T1竞品是否已覆盖 + 同类视频平均点赞 + 评论区需求度
- **人格匹配度**: 与 SOUL.md 核心信念的重叠度（自动计算）

信号注入 topic_pool.json：
```json
{
  "id": "topic-001",
  "title": "抑郁症是文化病",
  "competitor_signal": {
    "similar_videos_count": 12,
    "avg_likes": 8500,
    "top_accounts_covered": false,
    "comment_demand_score": 0.82
  },
  "persona_fit_score": 0.95
}
```

### Phase 3: persona 人格化重写 ⭐

**三步迁移法（全网独家方法 — DeepResearch 确认无同类方案）：**

```
竞品爆款骨架 → 剥离竞品人格层 → 注入SOUL.md世界观 → STYLE.md风格把关 → 人格化重写
```

**具体步骤：**

1. **提取纯结构骨架** — 从 godtier 研究长文中提取观点骨架，去掉所有"AI腔"
2. **世界观注入** — SOUL.md 的核心信念作为 system prompt 约束
3. **风格指紋注入** — STYLE.md 15维控制：开头模式、括号使用、破折号密度、反AI特征≥7/10、情感温度由 persona/STYLE.md 确定（禁止硬编码2/10）、结尾行动号召/诗意/质问
4. **🆕 标题生成（竞品规律驱动）** — 基于 Phase 0 竞品蒸馏的「维度6：爆款标题规律」，套用已验证的高效句式（"不是A，是B"/冒号二元对照/引号反讽/身份词前置），不得凭感觉写
5. **🆕 摘要生成** — 100-150字，公众号 feed 流显示用。必须包含具体数字/场景/人物，禁止抽象概括
6. **风格后验** — 对照 STYLE.md 检查15维通过率

**质量门禁：**
- 反AI特征 ≥ 7/10
- 人格盲测 "这像你写的" > 70%
- 无连续3句与竞品相同
- 🆕 标题符合竞品规律（句式可追溯），长度 8-35 字
- 🆕 摘要存在且 > 50 字，含具体场景/数字，非抽象概括

### Phase 3.5: QA 长文质量保障 ⭐ 新增 🔴 强制门禁

> 在 persona 重写产出的每篇长文上强制执行。跳过即违规。

**四步流程：**

```
persona 重写产出长文
       │
       ▼
步骤1: 搜证 ─── 任何事实性修改前，先 WebSearch 收集真实数据
  ├── 历史事件 → 时间/人物/规模/结果
  ├── 政策法规 → 准确名称(含"试行"等后缀)/发布时间/发布机构
  ├── 统计数据 → 具体数字/来源/口径/年份
  └── 案例引用 → 是否真实存在
       │
       ▼
步骤2: 逻辑一致性审查 ─── 检测"结构/个体矛盾"
  ├── 前文证明了"个体行动被系统惩罚" → 后文不能给"个体行动清单"
  ├── 前文展示了"结构性问题" → 后文不能以"敢不敢想象"结尾
  ├── 故事人物的处境与给他们的建议必须自洽
  └── 结尾诚实度 > 情绪感染力
       │
       ▼
步骤3: L4 防幻觉验证
  ├── claim_extractor.py → 提取四类声明
  ├── Claude 逐条 WebSearch
  ├── fact_checker.py → 关键词匹配判定（第一道筛选）
  └── Claude 语义覆盖（Python PASS 不代表事实正确）
       │
       ▼
步骤4: 迭代修复
  ├── FALSIFIED → 立即修正 → 重跑验证
  ├── 数据精度问题 → 补充来源/修正数字/区分统计口径
  └── 3次修复仍 FAIL → 选题标记"事实不可验证·放弃"（默认值，可调整）
```

**触发规则：**

| 触发条件 | 执行范围 |
|---------|---------|
| persona 重写产出新长文 | 完整四步 |
| 对已有长文做事实性修改 | 搜证 → 逻辑审查 → L4 |
| 对已有长文做结构性重写 | 搜证 → 逻辑审查 → L4 |
| Phase 3.5 建立前产出的存量文章首次修改 | 完整四步 QA |
| 仅修改错别字/格式 | 跳过（发现明显事实错误时标记并触发 QA） |

**逻辑审查自查清单：**
1. 核心论点的维度是结构性还是个体性？
2. 结尾方案与论点维度是否一致？
3. 有没有"前文证明某条路走不通，后文建议走这条路"？
4. 结尾是诚实承认困难，还是假装有简单答案？
5. 故事人物的处境与给他们的建议是否自洽？

**常见反模式：**

| 文章类型 | ❌ 错误结尾 | ✅ 正确方向 |
|---------|-----------|-----------|
| 劳动/阶级批判 | "你只需要勇敢一点/算清楚账/早点下班" | "一个人做不到。找到和你处境一样的人。" |
| 系统性问题 | "只要我们还疼/愤怒/记住，一切就有可能" | "疼是信号。但信号需要被连接才能变成力量。" |
| 个体觉醒 | "取决于老赵。取决于陈姐。" | "取决于老赵能不能找到其他老赵。" |

### Phase 4: 输出 — 微信长文

```
wechat/output/
├── wechat_articles/
│   ├── evergreen/            ← 常青内容
│   │   └── {title}_{date}.md
│   └── hot/                  ← 热点内容
│       └── {title}_{date}.md
└── briefs/                   ← 手动模式采访简报
    └── {topic}_brief_{date}.md
```

每篇微信长文输出格式：
- 标题 + 摘要 + 作者（被压迫者小组）
- 元数据头：生成日期、persona 引擎版本、SOUL+STYLE+PERSONA 注入标记
- 正文：场景化开头 → 数据/论证 → 结构性分析 → 开放式结尾
- 情感温度：读取 persona/STYLE.md（禁止硬编码固定值）

📄 如需将微信长文拆分为抖音/B站脚本 → 交给 [douyin 项目](https://github.com/r-ayin/douyin)（独立仓库）

## 🔴 管线强制执行（v2.3 — 脚本门禁）

> 每个阶段必须通过 `scripts/pipeline-gate.sh` 门禁。
> 门禁是 shell 脚本——子 Agent 无法跳过，主 LLM 必须调用。
> 
> ```bash
> bash scripts/pipeline-gate.sh check {phase}  # 进阶段前验证前置checkpoint
> bash scripts/pipeline-gate.sh verify {phase}  # 出阶段后验证产出质量
> bash scripts/pipeline-gate.sh status           # 查看管线状态
> ```
> 
> 门禁不通过 = 阻断执行。这不是建议，是 exit 2/exit 3 硬阻断。

### 竞品搜索规则

搜公众号文章**必须用搜狗微信搜索**，不用 Google/Bing/WebSearch：

```
https://weixin.sogou.com/weixin?type=2&query={关键词}
```

Google/Bing 对 `mp.weixin.qq.com` 收录极差。搜狗是微信生态内搜索引擎，直接索引公众号内容。

**竞品标题研究（Phase 0 必做）**：搜狗搜索完成后，额外搜一轮标题关键词，提炼爆款标题的句式规律（"不是A，是B"/冒号二元对照/引号反讽/身份词前置/情绪三件套），输出到 competitor-style 报告的「维度6：爆款标题规律」。Phase ③ persona 重写时必须基于此规律生成标题，不得凭感觉写标题。

```
发布微信长文 → 阅读量/在看/分享/留言
  → topic_feedback(reward) → 选题排序自动更新
  → style_feedback(signal) → 累积≥5条 → STYLE.md 进化周期
  → verification_feedback → FALSIFIED 记录 → 选题库标注"事实风险"
```

### QA 反馈

| 信号 | 触发条件 | 行动 |
|------|---------|------|
| 长文首次 QA PASS | FALSIFIED=0 + 逻辑一致性通过 | 正常发布 |
| 修复后 PASS | retry≥1 后通过 | 记录该选题为"事实高风险" |
| 3次修复后仍 FAIL | retry=3 仍 FALSIFIED>0 | 选题标记为"不可验证·放弃"，从选题池移除（默认值，可调整） |

## 五、选题池管理

### 选题生命周期

```
信号出现 → 进入选题池 → 排序打分 → 选中开发 → QA通过 → 发布 → 数据反馈 → 归档/复盘
```

### 选题池文件

```
douyin/
├── topic-pool/
│   ├── hot-pool.json       ← 热点选题池 (按UCB+对标+人格排序)
│   ├── evergreen-pool.json ← 常青选题池 (按重要性+完成度排序)
│   ├── series-plan.json    ← 系列规划 (季度主线+每篇进度)
│   └── published-log.json  ← 已发布记录 (日期/类型/数据)
```

### 选题优先级公式

```
选题优先级 = 受众痛点(0-10) × 信息差程度(0-10) × 人格匹配度(0-1) × 时效窗口(0-1)
```

| 维度 | 评分标准 |
|------|---------|
| 受众痛点 | 10=几乎所有人都在面对的问题 |
| 信息差程度 | 10=普通人完全不了解但严重影响他们的机制 |
| 人格匹配度 | 与SOUL.md核心信念的重叠度（自动计算） |
| 时效窗口 | 1=现在就可以写 / 0.5=需要等事件 / 0=已过时 |
